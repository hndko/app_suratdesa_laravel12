# PRODUCT REQUIREMENTS DOCUMENT (PRD)

**Nama Produk:** SIMADES (Sistem Informasi Desa)

**Klien:** Pemerintah Desa
**Pengembang:** Mari Partner

## 1. Latar Belakang & Tujuan Produk

Pemerintah Desa memerlukan platform digital terintegrasi untuk meningkatkan kualitas pelayanan publik dan transparansi administrasi sejalan dengan program transformasi digital dari Kementerian Desa PDTT.

Tujuan utama sistem ini adalah:

1. Meningkatkan efisiensi pelayanan administrasi (khususnya persuratan).

2. Mempermudah pengelolaan data kependudukan secara terpusat.

3. Menyediakan wadah pengaduan dan pusat informasi yang cepat bagi masyarakat.

_(Catatan: Modul Manajemen Keuangan dan Laporan ditunda/di-skip pada fase pengembangan tahap pertama ini)._

## 2. Target Pengguna (User Personas)

Sistem ini dirancang untuk mengakomodasi 4 jenis pengguna utama:

- **Masyarakat (Warga):** Pengguna publik tanpa akses _login_. Menggunakan sistem untuk melihat pengumuman, mengajukan surat secara _online_ via NIK, dan membuat laporan pengaduan.
- **Operator Desa (Staf):** Perangkat desa yang bertugas mengelola data kependudukan (CRUD), memverifikasi dan mencetak ajuan surat warga, serta membalas pengaduan.
- **Kepala Desa:** Pengguna level eksekutif. Fokus pada pemantauan metrik, analitik _dashboard_, dan persetujuan akhir (jika diperlukan).
- **Super Admin:** Tim IT yang memiliki kontrol penuh untuk mengatur hak akses (_role & permissions_), konfigurasi _website_, dan integrasi API (Fonnte).

## 3. Kebutuhan Fungsional (Functional Requirements)

### 3.1.

Modul Autentikasi & Otorisasi

- **FR-1.01:** Sistem harus menyediakan halaman _login_ untuk perangkat desa.
- **FR-1.02:** Sistem harus mendukung _Role-Based Access Control_ (RBAC) dinamis untuk membatasi akses menu berdasarkan peran (Admin, Operator, Kades).
- **FR-1.03:** Sistem harus mencatat riwayat aktivitas (_Activity Log_) pengguna di dalam _backend_ untuk kebutuhan audit.
- **FR-1.04:** Sistem menyediakan AI Adapter Gateway untuk mengelola provider AI dan log penggunaan AI.

### 3.2.

Modul Dashboard & Analitik

- **FR-2.01:** Menampilkan ringkasan total penduduk, jumlah Kepala Keluarga (KK), dan rasio jenis kelamin.
- **FR-2.02:** Menampilkan grafik analitik harian/bulanan untuk pengajuan surat dan status pengaduan masyarakat.

### 3.3.

Modul Sistem Data Kependudukan

- **FR-3.01:** Sistem harus bisa mengelola (CRUD) data Kartu Keluarga (KK).
- **FR-3.02:** Sistem harus bisa mengelola (CRUD) data anggota keluarga/individu yang tertaut dengan suatu nomor KK.
- **FR-3.03:** Sistem harus menyimpan atribut lengkap sesuai KTP (NIK, Nama, TTL, Agama, Pendidikan, Pekerjaan, Status Kawin, Golongan Darah, SHDK).
- **FR-3.04:** Menyediakan fungsionalitas pencarian data penduduk berdasarkan NIK atau Nama.
- **FR-3.05:** Sistem mendukung import data penduduk dan KK dari Excel/CSV dengan preview validasi.

### 3.4.

Modul Pelayanan Pembuatan Surat Online

- **FR-4.01:** Masyarakat dapat mengajukan surat secara mandiri melalui _frontend_ dengan memasukkan NIK dan keperluan, tanpa perlu _login_ ke sistem.
- **FR-4.02:** Sistem memvalidasi NIK masyarakat ke database kependudukan.
- **FR-4.03:** Sistem menyediakan manajemen untuk 15 _template_ surat standar desa.
- **FR-4.04:** Sistem secara otomatis mengisi (_auto-fill_) data diri warga ke dalam _template_ surat yang dipilih.
- **FR-4.05:** Admin/Operator dapat mengubah status pengajuan surat (Menunggu, Diproses, Selesai).
- **FR-4.06:** Sistem mendukung approval bertingkat: menunggu, diproses, diverifikasi, disetujui, selesai, atau ditolak.
- **FR-4.07:** Surat yang disetujui/selesai memiliki QR/kode verifikasi publik terbatas.
- **FR-4.08:** Operator dapat meminta saran AI untuk redaksi dan validasi placeholder template surat.

### 3.5.

Modul Pengaduan Masyarakat

- **FR-5.01:** Masyarakat dapat mengirimkan formulir pengaduan (Judul, Kategori, Isi, Lampiran Foto) melalui _frontend_ tanpa harus _login_.
- **FR-5.02:** Sistem memberikan Nomor Tiket/Kode unik kepada pelapor untuk melacak status aduan.
- **FR-5.03:** Operator dapat membaca, memfilter berdasarkan kategori/status, dan memberikan tanggapan balasan pada laporan masuk.
- **FR-5.04:** Operator dapat meminta AI menganalisis pengaduan untuk ringkasan, kategori rekomendasi, prioritas, dan draft balasan.

### 3.6.

Modul Informasi & Pengumuman Desa

- **FR-6.01:** Admin dapat membuat, mengedit, dan menghapus artikel/pengumuman (CMS sederhana).
- **FR-6.02:** Pengumuman dengan status _Published_ akan tampil di halaman _Landing Page_ masyarakat.

### 3.7.

Modul Integrasi Notifikasi (Opsional)

- **FR-7.01:** Sistem terintegrasi dengan Fonnte API menggunakan nomor WhatsApp klien.
- **FR-7.02:** Sistem mengirimkan pesan WhatsApp otomatis kepada warga ketika status pengajuan surat berubah menjadi "Selesai".

## 4. Kebutuhan Non-Fungsional (Non-Functional Requirements)

- **NFR-1 (UI/UX):** Antarmuka wajib responsif (_Mobile_ & _Desktop_ friendly), menggunakan _custom styling_ (Tailwind/Bootstrap manual) tanpa UI Builder otomatis.

- **NFR-2 (Keamanan):** Sandi pengguna dienkripsi secara _hash_ (Bcrypt/Argon2). Terdapat proteksi CSRF pada setiap _form submission_. Data penduduk tidak diekspos sembarangan ke _frontend_.

- **NFR-3 (Performa):** _Load time_ halaman _dashboard_ dan _frontend_ ditargetkan di bawah 2 detik.

## 5. Spesifikasi Teknologi (Tech Stack)

- **Bahasa Pemrograman:** PHP 8.2+
- **Framework Backend:** Laravel 12 (Core, Routing, Eloquent ORM)
- **Framework Frontend:** Blade Templating, Manual Layouting (tanpa Breeze/Jetstream starter kit)
- **Template Admin:** AdminLTE 3/Bootstrap 4 (Integrasi Manual)
- **Database:** MySQL
- **Package Utama:** \* `spatie/laravel-permission` (RBAC)
- `spatie/laravel-activitylog` (Audit)
- `laravel-lang/common` (Lokalisasi Bahasa Indonesia)

## 6. Asumsi dan Batasan (Out of Scope)

1. Modul Manajemen Keuangan tidak dimasukkan dalam _deliverables_ pengembangan saat ini.

2. Sistem _frontend_ (_Landing Page_ masyarakat) dibuat fungsional secara fitur, fokus pada kecepatan dan minimalis, bukan desain animasi berat.
3. Klien desa akan menyediakan data _master_ penduduk (Excel/CSV) untuk diimpor, atau melakukan input manual di awal jika fitur impor belum siap.
