## рџ”ђ Konfigurasi `.env` Lokal

Contoh konfigurasi lokal:

```env
APP_NAME="SIMADES"
APP_ENV=local
APP_DEBUG=true
APP_URL=http://127.0.0.1:8000
APP_VERSION=v3.0.48

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=db_suratdesa_laravel12
DB_USERNAME=root
DB_PASSWORD=

QUEUE_CONNECTION=database
FILESYSTEM_DISK=public
FONNTE_TOKEN=
DEFAULT_ADMIN_PASSWORD=password-yang-kuat
```

Catatan penting:

- Jangan commit `.env`.
- Jangan mengganti `APP_KEY` setelah data AI provider dibuat, karena API key AI memakai encryption Laravel.
- Isi `FONNTE_TOKEN` hanya jika ingin menguji notifikasi WhatsApp.
- Isi `DEFAULT_ADMIN_PASSWORD` sebelum seed awal agar akun default tidak memakai password lemah.

## рџ“¬ Queue Lokal

## рџ¤– AI Gateway Lokal

Provider AI diatur dari backend:

```text
AI Provider Gateway
```

Hal yang perlu diperhatikan:

- API key disimpan terenkripsi di database.
- `APP_KEY` tidak boleh berubah setelah API key tersimpan.
- Custom provider dapat memakai `base_url`, `api_key`, `model`, `temperature`, `max_tokens`, `timeout`, dan `retry`.
- Jika test koneksi gagal, cek base URL, model, API key, koneksi internet, dan log AI Usage Logs.

## рџ”ђ Permission & Role

Setelah menambah fitur, route backend, tombol aksi, export, upload, validasi khusus, request penting, integrasi, atau permission baru:

```bash
php artisan simades:sync-permissions
```

Semua permission wajib dicatat di:

```text
database/seeders/RolePermissionSeeder.php
```

Ini penting agar role baru yang dibuat client tetap bisa diberi akses fitur secara lengkap.

## рџ§Ї Troubleshooting Lokal

| Masalah | Solusi |
| --- | --- |
| Halaman putih/error 500 | Cek `storage/logs/laravel.log`, pastikan `.env`, database, dan `APP_KEY` benar. |
| Upload tidak tampil | Jalankan `php artisan storage:link`, cek `FILESYSTEM_DISK=public`. |
| Permission tidak berubah | Jalankan `php artisan simades:sync-permissions` dan `php artisan permission:cache-reset`. |
| Config lama masih terbaca | Jalankan `php artisan config:clear` atau `php artisan optimize:clear`. |
| View lama masih tampil | Jalankan `php artisan view:clear`. |
| WhatsApp tidak terkirim | Jalankan queue worker, cek `FONNTE_TOKEN`, cek log job. |
| AI test gagal | Cek provider aktif, API key, base URL, model, timeout, dan AI Usage Logs. |

## рџ“ќ Catatan Penting

- Jangan menjalankan `migrate:fresh --seed` pada database yang berisi data penting.
- Jangan mengganti `APP_KEY` pada environment yang sudah menyimpan API key AI.
- Jangan commit `.env`, file upload user, atau backup database.
- Setelah update v3, jalankan migration dan sync permission sebelum menguji AI Gateway, import penduduk, approval surat, QR verifikasi, dan setting portal publik.
