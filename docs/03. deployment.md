# 🚢 Panduan Deployment SIMADES

Dokumen ini berisi panduan deployment SIMADES untuk **shared hosting** dan **VPS**. Sesuaikan path, user, versi PHP, domain, dan fitur panel hosting yang tersedia.

Asset UI SIMADES memakai file statis di `public/assets`, sehingga deployment normal tidak membutuhkan `npm install` atau `npm run build`.

## ✅ Checklist Umum Production

- `APP_ENV=production`
- `APP_DEBUG=false`
- `APP_URL` memakai domain HTTPS.
- `APP_KEY` sudah dibuat dan tidak berubah setelah go-live.
- Database production sudah dibuat dan kredensialnya benar.
- `DEFAULT_ADMIN_PASSWORD` diisi password kuat sebelum seeding pertama.
- `FONNTE_TOKEN` diisi jika WhatsApp aktif.
- `QUEUE_CONNECTION=database` atau driver queue lain yang tersedia.
- `FILESYSTEM_DISK=public`
- Folder `storage/` dan `bootstrap/cache/` writable.
- Domain diarahkan ke folder `public/`.
- `storage:link` sudah tersedia.
- Queue worker, Supervisor, atau cron sudah disiapkan.
- Backup database dilakukan sebelum migration/update besar.

## 🔐 Contoh `.env` Production

```env
APP_NAME="SIMADES"
APP_ENV=production
APP_KEY=base64:ISI_DARI_KEY_GENERATE
APP_DEBUG=false
APP_URL=https://simades.example.com
APP_VERSION=v3.0.49

DB_CONNECTION=mysql
DB_HOST=127.0.0.1
DB_PORT=3306
DB_DATABASE=nama_database
DB_USERNAME=user_database
DB_PASSWORD=password_database

QUEUE_CONNECTION=database
FILESYSTEM_DISK=public
FONNTE_TOKEN=token-fonnte-jika-ada
DEFAULT_ADMIN_PASSWORD=password-admin-kuat
```

Penting:

- Jangan mengganti `APP_KEY` setelah production berjalan, terutama jika AI provider sudah disimpan.
- Jangan aktifkan `APP_DEBUG=true` di production.
- Jangan upload `.env` ke repository.

## 🌐 Deployment Shared Hosting

Shared hosting biasanya punya keterbatasan akses shell, process manager, dan document root. Target paling aman adalah domain/subdomain mengarah ke folder `public/`.

### Opsi A: Document Root Bisa Diubah

Struktur direkomendasikan:

```text
/home/user/simades/          # source Laravel
/home/user/simades/public/   # document root domain/subdomain
```

Atur document root domain ke:

```text
/home/user/simades/public
```

Lalu upload seluruh source project ke `/home/user/simades`.

Jika terminal tersedia:

```bash
cd /home/user/simades
composer install --no-dev --optimize-autoloader
cp .env.example .env
php artisan key:generate --force
php artisan migrate --force
php artisan db:seed --class=SettingSeeder --force
php artisan simades:sync-permissions
php artisan storage:link
php artisan optimize
```

Jika ini deploy pertama dan ingin seed lengkap:

```bash
php artisan migrate --seed --force
php artisan simades:sync-permissions
```

### Opsi B: Document Root Tidak Bisa Diubah

Jika domain wajib mengarah ke `public_html`, gunakan struktur:

```text
/home/user/simades/       # source Laravel di luar public_html
/home/user/public_html/   # isi folder public Laravel
```

Langkah:

1. Upload semua source Laravel ke `/home/user/simades`.
2. Pindahkan isi folder `/home/user/simades/public` ke `/home/user/public_html`.
3. Edit `/home/user/public_html/index.php`.
4. Sesuaikan path:

```php
require __DIR__.'/../simades/vendor/autoload.php';
$app = require_once __DIR__.'/../simades/bootstrap/app.php';
```

5. Pastikan `.env` ada di `/home/user/simades/.env`.
6. Jalankan command Laravel dari `/home/user/simades` jika terminal tersedia.

### Jika Shared Hosting Tidak Ada SSH

Pilihan yang bisa dilakukan:

- Jalankan `composer install --no-dev --optimize-autoloader` di lokal dengan versi PHP yang sama.
- Upload folder `vendor/`, source project, `.env`, dan isi `public/`.
- Jalankan migration melalui fitur terminal panel jika ada.
- Jika tidak ada fitur terminal sama sekali, minta provider menjalankan command artisan atau gunakan fitur cron one-time sementara.

Command minimal yang tetap perlu dijalankan:

```bash
php artisan migrate --force
php artisan simades:sync-permissions
php artisan storage:link
php artisan optimize
```

### Storage Link di Shared Hosting

Jika `php artisan storage:link` gagal karena symlink diblokir:

- Coba fitur symlink dari panel hosting.
- Minta provider mengaktifkan symlink.
- Alternatif terakhir, gunakan route/controller untuk menyajikan file storage, namun ini perlu implementasi khusus dan tidak direkomendasikan jika symlink tersedia.

### Cron Queue di Shared Hosting

Jika tidak ada Supervisor, gunakan cron worker pendek:

```cron
* * * * * cd /home/user/simades && php artisan queue:work --stop-when-empty --tries=3 --timeout=60 >> /dev/null 2>&1
```

Jika memakai scheduler:

```cron
* * * * * cd /home/user/simades && php artisan schedule:run >> /dev/null 2>&1
```

## 🖥️ Deployment VPS

VPS memberi kontrol penuh. Contoh ini memakai Ubuntu, Nginx, PHP-FPM, MySQL/MariaDB, dan Supervisor.

### 1. Install Kebutuhan Server

```bash
sudo apt update
sudo apt install nginx mysql-server unzip git supervisor
sudo apt install php8.2-fpm php8.2-cli php8.2-mysql php8.2-mbstring php8.2-xml php8.2-curl php8.2-zip php8.2-bcmath php8.2-gd
```

Install Composer jika belum ada:

```bash
cd /tmp
php -r "copy('https://getcomposer.org/installer', 'composer-setup.php');"
php composer-setup.php
sudo mv composer.phar /usr/local/bin/composer
composer --version
```

### 2. Clone Project

```bash
sudo mkdir -p /var/www
sudo chown -R $USER:$USER /var/www
git clone https://github.com/hndko/app_suratdesa_laravel12.git /var/www/simades
cd /var/www/simades
```

### 3. Install Dependency

```bash
composer install --no-dev --optimize-autoloader
cp .env.example .env
php artisan key:generate --force
```

Edit `.env` production:

```bash
nano .env
```

### 4. Setup Database

```bash
sudo mysql
```

Contoh SQL:

```sql
CREATE DATABASE simades CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE USER 'simades_user'@'localhost' IDENTIFIED BY 'password-kuat';
GRANT ALL PRIVILEGES ON simades.* TO 'simades_user'@'localhost';
FLUSH PRIVILEGES;
EXIT;
```

Update `.env` sesuai database.

### 5. Migration, Seeder, Storage, Cache

Deploy pertama:

```bash
php artisan migrate --seed --force
php artisan simades:sync-permissions
php artisan storage:link
php artisan optimize
```

Deploy update:

```bash
php artisan migrate --force
php artisan simades:sync-permissions
php artisan optimize
```

### 6. Permission Folder

```bash
sudo chown -R www-data:www-data /var/www/simades
sudo chmod -R ug+rwx /var/www/simades/storage /var/www/simades/bootstrap/cache
```

### 7. Nginx

Buat config:

```bash
sudo nano /etc/nginx/sites-available/simades
```

Isi contoh:

```nginx
server {
    listen 80;
    server_name simades.example.com;
    root /var/www/simades/public;

    index index.php index.html;

    add_header X-Frame-Options "SAMEORIGIN";
    add_header X-Content-Type-Options "nosniff";

    charset utf-8;

    location / {
        try_files $uri $uri/ /index.php?$query_string;
    }

    location = /favicon.ico { access_log off; log_not_found off; }
    location = /robots.txt  { access_log off; log_not_found off; }

    error_page 404 /index.php;

    location ~ \.php$ {
        include snippets/fastcgi-php.conf;
        fastcgi_pass unix:/run/php/php8.2-fpm.sock;
        fastcgi_param SCRIPT_FILENAME $realpath_root$fastcgi_script_name;
        include fastcgi_params;
    }

    location ~ /\.(?!well-known).* {
        deny all;
    }
}
```

Aktifkan:

```bash
sudo ln -s /etc/nginx/sites-available/simades /etc/nginx/sites-enabled/simades
sudo nginx -t
sudo systemctl reload nginx
```

### 8. HTTPS

Gunakan Certbot:

```bash
sudo apt install certbot python3-certbot-nginx
sudo certbot --nginx -d simades.example.com
```

Pastikan `.env`:

```env
APP_URL=https://simades.example.com
```

Lalu:

```bash
php artisan optimize:clear
php artisan optimize
```

### 9. Supervisor Queue

Buat config:

```bash
sudo nano /etc/supervisor/conf.d/simades-queue.conf
```

Isi:

```ini
[program:simades-queue]
process_name=%(program_name)s_%(process_num)02d
command=php /var/www/simades/artisan queue:work --sleep=3 --tries=3 --timeout=60
autostart=true
autorestart=true
stopasgroup=true
killasgroup=true
user=www-data
numprocs=1
redirect_stderr=true
stdout_logfile=/var/www/simades/storage/logs/queue.log
stopwaitsecs=3600
```

Aktifkan:

```bash
sudo supervisorctl reread
sudo supervisorctl update
sudo supervisorctl restart simades-queue:*
sudo supervisorctl status
```

Sebelum update:

```bash

```


## 🔐 Update Permission di Production

Gunakan setelah ada perubahan fitur, tombol, route, role, export, upload, AI, import, atau approval:

```bash
php artisan simades:sync-permissions
```

Catatan:

- Permission baru akan dibuat.
- Permission lama sebaiknya tidak langsung dihapus jika sudah production.
- Role custom client tidak boleh diasumsikan selalu sama dengan role default.
- Jika client mengganti nama role, pastikan role tersebut tetap diberi permission yang sesuai dari UI Role & Permission.

## ↩️ Rollback Ringkas

Jika update gagal:

1. Aktifkan maintenance:

```bash
php artisan down
```

2. Kembalikan kode ke commit sebelumnya:

```bash
git log --oneline -5
git checkout COMMIT_SEBELUMNYA
composer install --no-dev --optimize-autoloader
```

3. Restore database dari backup jika migration sudah mengubah struktur/data.

4. Bersihkan cache dan restart queue:

```bash
php artisan optimize:clear
php artisan optimize
php artisan queue:restart
php artisan up
```

## 🧯 Troubleshooting Production

| Masalah | Solusi |
| --- | --- |
| Error 500 | Cek `storage/logs/laravel.log`, permission folder, `.env`, dan koneksi DB. |
| CSS/JS/gambar tidak tampil | Pastikan domain mengarah ke `public/`, asset ada di `public/assets`, dan `APP_URL` benar. |
| Upload tidak tampil | Jalankan `php artisan storage:link`, cek symlink dan permission storage. |
| Login/permission bermasalah | Jalankan `php artisan simades:sync-permissions` dan `php artisan permission:cache-reset`. |
| Setting sudah disimpan tapi frontend belum berubah | Jalankan `php artisan optimize:clear`; pastikan cache setting sudah clear dari proses update. |
| Queue tidak jalan | Cek Supervisor/cron, `php artisan queue:failed`, dan `storage/logs/queue.log`. |
| WhatsApp gagal | Cek `FONNTE_TOKEN`, queue, nomor tujuan, dan log Laravel. |
| AI gagal | Cek provider aktif, API key, base URL, model, timeout/retry, dan AI Usage Logs. |
